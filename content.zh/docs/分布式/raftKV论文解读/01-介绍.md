# 介绍

## Paxos算法痛点

- 算法本身复杂、难理解
- 算法结构难以进行二次开发和直接利用

## Raft算法的提出

- 容易理解（算法分解 / 模块化：领导人选举、日志复制、安全）
- 容易进行二次开发

**特性：**（和其他一致性算法相比）
- 强领导者：角色具备更强的领导能力
- 领导选举：随机计时器 + 心跳机制
- 成员关系调整：共同一致的方法（使得 处于变化/调整的两个集群会有大多数机器的重叠 -> 使得 集群在成员变换的时候还可以工作）
